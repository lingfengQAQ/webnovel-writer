## 📖 快速开始

### 第一步：初始化项目

```bash
/webnovel-init
```

系统将引导您：
1. 选择题材（修仙/都市/玄幻/游戏/科幻）
2. 输入小说标题
3. 输入主角姓名
4. 自动生成项目结构

**生成的项目结构**:
```
webnovel-project/
├── .webnovel/
│   ├── state.json              # 动态状态管理（自动维护）
│   └── backups/
├── 设定集/
│   ├── 世界观.md               # 根据题材模板生成
│   ├── 力量体系.md
│   ├── 主角卡.md
│   └── 角色库/
├── 大纲/
│   └── 总纲.md                 # 卷-篇-章三层结构
├── 正文/
└── 审查报告/
```

### 第二步：规划大纲（可选）

```bash
/webnovel-plan 1    # 规划第 1 卷详细大纲
```

系统将：
1. 读取总纲框架
2. 细化第 1 卷的篇章结构
3. 生成 `大纲/第1卷-详细大纲.md`

### 第三步：开始创作

```bash
/webnovel-write 1    # 创作第 1 章
/webnovel-write 2    # 创作第 2 章
...
```

系统将：
1. 读取章节大纲和设定
2. **Think 步骤**规划爽点
3. 生成 3000-5000 字正文
4. 自动检查三大定律
5. 标记 [NEW_ENTITY]（如有）
6. 保存到 `正文/第0001章.md`

### 第四步：质量审查

每写完 10 章，系统自动触发审查。也可手动触发：

```bash
/webnovel-review 1-10    # 审查第 1-10 章
```

**5 个审查员**会检查：
1. 爽点密度（high-point-checker）
2. 设定一致性（consistency-checker）
3. 节奏控制（pacing-checker）
4. 人物 OOC（ooc-checker）
5. 情节连贯性（continuity-checker）

生成报告：`审查报告/第001-010章审查报告.md`

### 第五步：设定查询

随时查询设定信息：

```bash
/webnovel-query 主角           # 查询主角信息
/webnovel-query 筑基期         # 查询力量体系
/webnovel-query 血煞门         # 查询势力信息
/webnovel-query 未回收伏笔     # 查询待回收伏笔
```

## 🎯 核心特性

### 1. 防幻觉三大定律

#### 定律 1: 大纲即法律
✅ 不得擅自偏离已确认的大纲
⚠️ 如需偏离，标记 `[OUTLINE_DEVIATION]` 并说明

#### 定律 2: 设定即物理
✅ 角色实力/招式/物品必须符合设定集
⚠️ 发现冲突立即标记 `[POWER_CONFLICT]`

#### 定律 3: 发明需申报
✅ 所有新角色/地点/物品标记 `[NEW_ENTITY]`
✅ 系统自动提取并询问是否加入设定集

### 2. 爽点引擎

**Think 步骤强制规划**:
每章生成前，AI 必须先规划：
- 本章爽点类型（打脸/升级/收获/装逼）
- 铺垫-爆发-余韵节奏
- 预期字数

**爽点密度要求**:
- 每章至少 1 个爽点
- 每 5 章至少 1 个大爽点
- 每 10 章至少 1 次实力提升

### 3. 动态状态管理

**state.json 实时追踪**:
```json
{
  "protagonist_state": {
    "power": {"realm": "筑基期", "layer": 7},
    "location": {"current": "天云宗外门"},
    "golden_finger": {"cooldown": "剩余 3 天"}
  },
  "relationships": {
    "李雪": {"affection": 85, "status": "暧昧期"},
    "血煞门": {"hatred": 100, "status": "不死不休"}
  },
  "plot_threads": {
    "foreshadowing": [
      {"content": "林家宝库神秘铭文", "status": "待回收"}
    ]
  }
}
```

### 4. 交互式创作流

#### 场景 A: 剧情分支选择
当主角面临重要决策时，系统会暂停并询问：

```
🎭 检测到剧情分支点

当前情节: 血煞门少主挑衅主角...

请选择主角反应：
A. 当场斩杀（冲动路线）
B. 隐忍暂避（理智路线）
C. 自定义...
```

#### 场景 B: 设定补全
当检测到设定缺失时：

```
⚠️ 设定缺失提示

剧情需要: 主角进入"黑风山脉秘境"
问题: 设定集中未记录此秘境

请补充设定：
1. 秘境内有哪些危险/机遇？
2. 秘境开启频率？
3. 秘境风险等级？
```

### 5. 五维质量审查

| 审查员 | 职责 | 输出 |
|-------|------|------|
| high-point-checker | 爽感指数评分 | 0-100 分 + 建议 |
| consistency-checker | 设定冲突检测 | ✅/❌ + 具体问题 |
| pacing-checker | 节奏拖沓检查 | 有效信息密度分析 |
| ooc-checker | 人物失真检测 | 角色行为是否符合人设 |
| continuity-checker | 连贯性检查 | 时间线/地点/状态延续 |

## 📚 文件说明

### 用户可编辑文件

**设定集/**（可手动编辑）:
- `世界观.md` - 世界背景、历史、地理
- `力量体系.md` - 境界划分、修炼方法
- `主角卡.md` - 主角详细信息
- `角色库/` - 所有角色卡

**大纲/**（可手动编辑）:
- `总纲.md` - 全书卷-篇-章框架
- `第X卷-详细大纲.md` - 每卷的详细大纲

**正文/**（AI 生成）:
- `第XXXX章.md` - 每章正文

### 系统维护文件

**.webnovel/**（建议不要手动编辑）:
- `state.json` - 动态状态数据
- `backups/` - 自动备份文件

## 💡 创作建议

### 1. 先规划后创作
- 不要急于写正文
- 先完善设定集和大纲
- 大纲越详细，AI 生成质量越高

### 2. 充分利用交互式功能
- 遇到分支点时认真选择
- 设定不足时及时补充
- 不要让 AI 瞎编

### 3. 定期审查
- 每 10 章必须审查一次
- 及时修正问题
- 避免后期大规模返工

### 4. 善用查询功能
- 不确定设定时先查询
- 避免前后矛盾
- 保持设定一致性

## ⚠️ 常见问题

### Q: AI 生成的内容与大纲不符？
A: 检查 SKILL.md 中的"定律 1"是否被违反，如有 `[OUTLINE_DEVIATION]` 标记，说明 AI 在申请偏离。

### Q: 章节中出现未定义的角色/地点？
A: 查找 `[NEW_ENTITY]` 标签，确认是否已加入设定集。

### Q: 审查报告提示"战力崩坏"？
A: 检查 `state.json` 中的实力数据是否正确，手动修正后重新生成章节。

### Q: 如何修改已生成的章节？
A: 直接编辑 `正文/第XXXX章.md` 文件即可（Markdown 格式）。

### Q: state.json 数据混乱怎么办？
A: 该文件由系统自动维护，如需手动修改请参考 JSON 格式规范。
